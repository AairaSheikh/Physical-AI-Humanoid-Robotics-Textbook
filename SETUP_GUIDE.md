# Physical AI & Humanoid Robotics Textbook - Setup Guide

## Project Status

This project is a Hackathon I submission for creating a comprehensive AI-native textbook for teaching Physical AI and Humanoid Robotics. The project includes:

- **Docusaurus-based interactive textbook** with 4 modules and 20+ chapters
- **RAG Chatbot** for intelligent Q&A about the textbook content
- **Authentication system** using better-auth
- **Content personalization** based on user background
- **Urdu translation support**

## Implementation Steps (Complete These Locally)

### Phase 1: Frontend (Docusaurus) Setup

1. **Clone the repository locally**
   ```bash
   git clone https://github.com/AairaSheikh/Physical-AI-Humanoid-Robotics-Textbook.git
   cd Physical-AI-Humanoid-Robotics-Textbook
   ```

2. **Create Docusaurus project**
   ```bash
   npx create-docusaurus@latest docusaurus classic
   cd docusaurus
   npm install
   ```

3. **Install additional dependencies**
   ```bash
   npm install react-markdown remark-gfm @docusaurus/plugin-client-redirects
   npm install axios @openai/api
   npm install tailwindcss postcss autoprefixer
   npx tailwindcss init -p
   ```

4. **Create module documentation structure**
   ```bash
   mkdir -p docs/module-1-ros2
   mkdir -p docs/module-2-digital-twin
   mkdir -p docs/module-3-isaac
   mkdir -p docs/module-4-vla
   ```

5. **Add module content files** (use content from Claude artifacts)
   ```bash
   # Create files for each module:
   # docs/module-1-ros2/intro.md
   # docs/module-1-ros2/fundamentals.md
   # docs/module-1-ros2/nodes-and-topics.md
   # docs/module-1-ros2/services.md
   # docs/module-1-ros2/urdf.md
   # ... and similar for other modules
   ```

### Phase 2: Backend (FastAPI) Setup

1. **Setup Python environment**
   ```bash
   cd backend
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. **Create backend structure**
   ```bash
   mkdir -p app/models
   mkdir -p app/api
   mkdir -p app/services
   mkdir -p scripts
   touch app/__init__.py
   touch app/main.py
   touch app/config.py
   touch app/models/chat.py
   touch app/services/rag.py
   touch app/services/embeddings.py
   ```

3. **Setup environment variables**
   ```bash
   # Create .env file
   cp .env.example .env
   # Fill in your API keys:
   # OPENAI_API_KEY=your_key
   # QDRANT_URL=your_qdrant_cloud_url
   # QDRANT_API_KEY=your_key
   # DATABASE_URL=your_neon_postgres_url
   ```

### Phase 3: Database Setup

1. **Create Neon Postgres database**
   - Go to neon.tech and create a free account
   - Create a new project and database
   - Copy the connection string to .env as DATABASE_URL

2. **Setup Qdrant Cloud**
   - Go to qdrant.tech and create a free account
   - Create a cluster for vector storage
   - Copy the URL and API key to .env

3. **Initialize database schema**
   ```bash
   python scripts/setup_db.py
   ```

### Phase 4: Integration

1. **Create ChatBot React component** (src/components/ChatBot/index.tsx)
2. **Create RAG service** (app/services/rag.py with OpenAI integration)
3. **Create API endpoints** (app/main.py with FastAPI)
4. **Integrate chatbot into Docusaurus** (src/pages/index.tsx)

### Phase 5: Deployment

1. **Deploy frontend to GitHub Pages**
   ```bash
   cd docusaurus
   npm run build
   npm run deploy
   ```

2. **Deploy backend to Vercel/Railway/Heroku**
   ```bash
   # Follow your chosen platform's deployment guide
   ```

## Key Technologies

- **Frontend**: Docusaurus 3, React 18, TypeScript, Tailwind CSS
- **Backend**: FastAPI, Python 3.10+, SQLAlchemy
- **Databases**: Neon Postgres (relational), Qdrant Cloud (vector)
- **AI/ML**: OpenAI APIs, LangChain
- **Auth**: Better-auth
- **Deployment**: GitHub Pages (frontend), Vercel/Railway/Heroku (backend)

## API Endpoints

- `POST /api/chat` - Send chat message to RAG chatbot
- `GET /api/chat/{conversation_id}` - Get conversation history
- `POST /api/auth/signup` - User registration with background info
- `POST /api/auth/signin` - User login
- `POST /api/content/translate` - Translate content to Urdu
- `POST /api/content/personalize` - Get personalized content
